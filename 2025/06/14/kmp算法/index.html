KMP算法中的next数组详解

这几天在学习编程算法的过程中了解到了KMP算法，发现了其在子串在主串匹配时的强大功能，代码很优美简洁干练且时间复杂度低,感觉到十分有趣。实际上，它在数据结构的地位如同EM算法(这个EM算法也非常有趣，它是K-means和GNN算法的基础和核心，有兴趣的童鞋也可以去了解认识一下。)在机器学习里的地位，因此我想借此谈一谈对于KMP算法的理解，其实你们也可以理解为我做学习的笔记和心得吧，也想借此分享一下我自己的观点。
此外，这篇文章引用了很多人的图片，下方参考文献已做备注，在这里说一声感谢，感谢你们的分享👍。

待处理的问题

首先在此还需要想KMP算法的三位创始人Knuth,Morris,Pratt致敬，他们的算法在当今社会中可谓是遍地都是，在此首先感谢他们的聪明才智。
实际上这一问题是针对于字符串的子串匹配问题，比如字符串"aaabaaaaabcb"查找"aaabc"这个子串，这个该怎么做呢?
其中最为简单想到的字符串匹配算法是暴力匹配(Brute Force)，如下图(在此感谢 NebulaGMY_萌新星云做的图):

实际上为了找到匹配的模式串"aaabc"，我们将模式串一遍遍和主串"aaabaaaaabcb"比较，如果有一次不对，就移动模式串向右移动一位，直到移动到最后主串最后尾巴为止(即二者的长度之差为移动长度),其本质上是暴力匹配。

其中next数组中每个下标对应存储的元素是包括该元素的公共前后缀的长度(部分资料认为的next数组不一样，实际上是包不包含该元素，本文认为这里是包含该元素计算的最大公共前后缀长度，下例和代码也按这个标准),其原因就在于它要尽可能利用模式串的已匹配信息,其思路有点类似于哈希表一样存储已经遍历的信息并来查询使用。因为是公共的前后缀长度，既然存在公共相同的长度，我们就可以利用这个公共的前后缀来跳过不匹配的部分，实际上是直接将模式串向右移动。

上图我们可以看到是e和c不匹配，因此我们要开始移动模式串(注意:在匹配过程中，一定是主串不动，模式串在不断右移进行匹配)，但是需要注意的是c的存在最长前后前后公共前缀ab，实际上我们可以利用这一点进行下图的平移

可以看出由蓝色的c跳转到红色的c，其中模式串移动的长度是的已匹配长度 - 最长公共前后缀长度，模式串前面的ab移动到后面的ab，刚好二者公用这样就利用了已知的信息进行匹配而无需一遍遍遍历模式串再一次次移动模式串，需要指出的是，这两图的移动是一次程序执行的时间，其中被跳过的步数是一定不匹配(因为移动后的前缀不匹配，整个模式串肯定都不匹配，因为考虑的是首先前缀先匹配的存在)，我们需要查找的是可能匹配的存在，在这里想起了贪心算法，哈哈。
在此，用严谨的语言来描述:

  1. 模式串移动的长度 = 主串于模式串公共已匹配长度 - 最长公共前后缀长度
  2. 模式串中第一个不匹配字符的下标->（跳转到）最长公共前后缀长度的下标，例如上述两图中蓝c下标是5跳转到2即即有next[5]=3，而3=最大公共前后缀长度

而为了进行上述算法的实现，我们需要引入next数组存储最大公共前后缀长度。其中next数组由四个性质很重要，这些为我个人总结:

  1. next[i]的值表示下标为i的字符前的字符串最长相等前后缀的长度，这个之前已经说了。
  2. 它表示该处字符不匹配时应该回溯到的字符的下标，实际上这个跟接下来的具体代码有关
  3. next[0]的特殊性，它的值依据代码编写一般设为-1或0,我们认为其没有意义即没有前后缀字符，不怎么考虑。在此基础上，除next[0]外，next[i]对应的值就是最长公共前后缀长度
  4. 关于next[i]数值，其数值不能为前缀的最长长度(可以理解不包括本身)，因为如果是最长长度，相当于没有移动，因而认为它等效于0。

注:为什么有些资料next数组是0,1开头，而有些next数组是-1,0开头?(对应性质3)
-1,0开头与0, 1开头的next数组本质是一样的。实际上，以0, 1开头的next数组就是以-1,0开头的next数组每一项加1得到的。出现这种情况的原因在于模式串起始的索引值：在程序中，一个数组的索引的起始值为0；然而在考试和书中给的模式串起始值是多从1开始。所以在考试中遇到的next数组通常是以0, 1开头；而一些程序或教程中的next数组是以-1, 0开头。

代码思路(重难点)

我认为其中你会手算next数组或者了解了KMP算法的整体运行过程,但是可能很大概率写不出来kmp算法,这才是KMP算法的重难点。
其中重难点是next数组的构建，其实我一开始想到的是不断遍历并计算每一个数组元素内对应的next数组的值,但是这种时间复杂度太大了为O(m^2),实际上它是非常简洁的，时间复杂度是O(m+n),其思想如下:
初始化条件：

next[0]=−1\text{next}[0] = -1 next[0]=−1

递推公式（j>0j > 0j>0 时）：

next[j]={k+1,if p[j]=p[k+1] 且 k=next[j−1]递归 k←next[k],if p[j]≠p[k+1] 且 k≥0−1,otherwise (k=−1)\text{next}[j] = \begin{cases} k + 1, & \text{if } p[j] = p[k+1] \text{ 且 } k = \text{next}[j-1] \\ \text{递归 } k \leftarrow \text{next}[k], & \text{if } p[j] \neq p[k+1] \text{ 且 } k \geq 0 \\ -1, & \text{otherwise } (k = -1) \end{cases} next[j]=⎩⎨⎧​k+1,递归 k←next[k],−1,​if p[j]=p[k+1] 且 k=next[j−1]if p[j]=p[k+1] 且 k≥0otherwise (k=−1)​

其思路我来大致具体说明一下,它是一个双指针+递推+回溯递推的思路:
其中我们遍历的每个next[j]每次最多增加1即k+1要么就是回归其对于的next下标值，其次模式串最后一位字符不影响结果不考虑，因而其实我们数组遍历的范围是从0到length-2。
为什么每次最多增加1，在这里，我们采用双指针的思路，这里使用j和k，k是前缀的指针，而j是模式串的指针。如果两个指针的下一个字符相同的话我们每次都向右遍历一次，其长度肯定是+1比如aba到abab。这里说起来可能抽象，我们举个例子来说一下，比如aba字符串(k指向第一个a,j指向第二个a，其长度是1,实际上由原来的0变为1)到abab字符串(在这里,k指向第一个b，j指向第二个b，字符相同因而+1),代码上是:next[j]=k=1后j+1，再是next[j]=k+1=2。实际上它是在前面公共前后缀的基础上不断更新的，这样无需每一次都遍历每个字符的公共前后缀，这就是不断优化计算的。
其中这里的难点是当不相等时，为什么是k=next[k]，实际上在赋值之后，我们会继续之后的循环，如果相等则在next[k]的基础上+1，如果不相等则继续将k按照之前已遍历的结果向前走而再重置为0重新计算(利用已知的数据减少时间复杂度)，这里其实运用的思路就是利用之前的数据递推回溯，回溯到更短的前缀继续尝试匹配。
在这里，我给出数学归纳法的总结:
情况 1：P[k] == P[j]
直接扩展公共前后缀，next[j+1] = k+1。显然正确。
情况 2：P[k] != P[j]
需要找到一个新的 k'，使得 P[0..k'-1] 是 P[..j-1] 的真后缀，且 P[k'] == P[j]。
由于 next[k] 是 P[0..k-1] 的最长公共前后缀长度，所以有：
P[0..next[k]-1] == P[k-next[k]..k-1]
而P[0..next[k]-1]是P[0..k-1]中(之前判断的最长公共前后缀)的次长公共前缀，我们会继续按上述判断该次长的公共前缀与后缀是否相同来判断程序是否进入情况1或者情况2

注解: 我们实际上是在利用已知信息不断递归缩小范围
因此，如果P[0..next[k]-1] == P[0..j-1、] 的一个次长公共前后缀，我们将在这个次长公共前后缀+1继而的到next值，如果不行则继续上述的操作(一直递归)，比如变成P[0..next[next[k]]-1]的前缀。
当然众所周知递归很重要的一点是需要有一个递归条件，因而其跳出条件就是k变为-1那么值就为最初的开头，至此值设置为0即没有公共前后缀。

该程序不断递归地检查 P[next[k]...] 是否等于 P[j]，直到 k = -1跳出递归，值设置为0。
在这里我举个例子abacababd,我们以d前面紧邻的b来计算，此时k=3，j=7，前缀是abac和后缀abab，见下图:

c和d不同，因而k需要回溯，回溯到k=next[k]这里next[k=3]=1，因而k会回溯到1(利用之前已经存在的公共前后缀),j不变，由此我们可以观察到ab和ab是公共前后缀(p[1] == p[7]✔️)，继而k=k+1=2,next[7]=2,完成这一轮的计算.过程见下图:

最后的代码如下:

    # 构建并获得next数组
    def get_next(self,p: str) -> list:
        """
        由模式串t求出next数组
        :param p: 模式字符串
        :return: next数组
        """
        length = len(p)
        next_arr = [0] * length
        next_arr[0] = -1  # 第一个字符前无字符串，给值-1
        j, k = 0, -1

        while j < length - 1:
            # k为-1或比较的字符相等时
            if k == -1 or p[j] == p[k]:
                j += 1
                k += 1
                next_arr[j] = k
            else:
                k = next_arr[k]

        return next_arr

    // java
    // 构建并获得next数组
    public static int[] getNext(String p) {
        int length = p.length();
        int[] next = new int[length];
        next[0] = -1;  // 第一个字符前无字符串，给值-1
        int j = 0, k = -1;

        while (j < length - 1) {
            // k为-1或比较的字符相等时
            if (k == -1 || p.charAt(j) == p.charAt(k)) {
                j++;
                k++;
                next[j] = k;
            } else {
                k = next[k];
            }
        }

        return next;
    }

关于最后的kmp搜寻算法，这个就跟我们之前所想无太大区别，不需要借此优化什么了。实际思路就是到哪个字符用它的next数组进行右移动，直到遍历完整个主串。
下述为kmp搜寻算法:

# Python-KMP搜寻算法
def kmp(s: str, t: str) -> int:
        len_s, len_t = len(s), len(t)
        if len_t == 0:
            return 0  # 空模式串默认匹配位置0
        if len_s < len_t:
            return -1

        next_arr = get_next(t)
        i, j = 0, 0

        while i < len_s and j < len_t:
            if j == -1 or s[i] == t[j]:
                i += 1
                j += 1
            else:
                j = next_arr[j]  # i不变，j回退,j回退相当于模式串有移动

        return i - len_t if j == len_t else -1

// Java-KMP搜寻算法
    public static int kmp(String s, String t) {
        int lenS = s.length(), lenT = t.length();
        if (lenT == 0) {
            return 0;  // 空模式串默认匹配位置0
        }
        if (lenS < lenT) {
            return -1;
        }

        int[] next = getNext(t);
        int i = 0, j = 0;

        while (i < lenS && j < lenT) {
            if (j == -1 || s.charAt(i) == t.charAt(j)) {
                i++;
                j++;
            } else {
                j = next[j];  // i不变，j回退
            }
        }

        return (j == lenT) ? (i - lenT) : -1;
    }

最后，引此上述代码并解释kmp利用next数组计算的全过程，我举了一个例子供大家思考，比如:
主串s = "ABABABC"，模式串t = "ABABC"
next数组：t = "ABABC" → next = [-1, 0, 0, 1, 2]
匹配过程：
初始：
i=0, j=0 → s[0]=A == t[0]=A → i=1, j=1

i=1, j=1 → s[1]=B == t[1]=B → i=2, j=2

i=2, j=2 → s[2]=A == t[2]=A → i=3, j=3

i=3, j=3 → s[3]=B == t[3]=B → i=4, j=4

i=4, j=4 → s[4]=A != C=t[4] → j = next[4] = 2 # 不匹配实际上j左移动等同于模式串向右移动，步长为原先的j-现在的j

i=4, j=2 → s[4]=A == t[2]=A → i=5, j=3

i=5, j=3 → s[5]=B == t[3]=B → i=6, j=4

i=6, j=4 → s[6]=C == t[4]=C → i=7, j=5（此时 j == len(t)，匹配成功）
其中我们函数最终返回为：i - len(t) = 7 - 5 = 2（即模式串在主串的起始下标为2）
